# 블로그 포스트 안티패턴 점검 스킬

발행 전 포스트(`_posts/**/*.md`)에서 LLM 글쓰기 함정을 점검한다. **자동 수정하지 않는다.** 라인 번호·인용·권장만 제시하고 결정은 사용자가 한다.

근거: `CLAUDE.md`의 "LLM 글쓰기 안티패턴 체크리스트" 섹션 + 메모리 `feedback_llm_antipatterns.md`.

## 대상 결정

1. 인자로 파일 경로가 주어지면 그 파일
2. 비어있으면 `ls -lt _posts/**/*.md`로 최근 수정된 포스트 중 `.en.md`/`.ja.md`를 제외한 한국어 원본 자동 선택
3. 모호하면 사용자에게 후보 3개 제시 후 선택 받기

## 점검할 6가지 패턴

1. **불릿 묶음 뒤 종합 문단 부재** — 3개 이상 항목 나열 후 "왜/어떻게" 문단 없이 다음 섹션 진입
2. **정보 진전 정체** — 같은 주장이 다른 표현으로 반복, 새 사실·수치·근거 추가 없음
3. **표·다이어그램 뒤 공허한 요약** — "이 모든 것이 자동으로 조합되어 동작합니다" 같은 일반화
4. **모호한 지시대명사** — "이/그/이러한"의 referent 불명, 특히 두 문장 이상 떨어진 경우
5. **입장 약화** — "~할 수 있다", "~인 경향이 있다" 헤지 남발, 1인칭 경험 사라짐
6. **카드·항목의 단조로운 리듬** — 모든 항목이 비슷한 길이라 중요도 차이가 안 드러남

## 검출 휴리스틱

각 패턴은 휴리스틱과 정성 평가를 결합한다. 휴리스틱이 약한 항목은 Claude의 정성 판단을 명시한다.

### #1 불릿 묶음 뒤 종합 문단 부재
- 3개 이상의 `- ` / `* ` / `\d+. ` 연속을 한 묶음으로 식별
- 묶음 종료 직후 다음 줄을 본다:
  - 빈 줄 + 헤딩(`#`/`##`/...) → **안티패턴** (종합 없이 다음 섹션)
  - 빈 줄 + 또 다른 불릿 묶음 → **안티패턴**
  - 빈 줄 + 50자 미만 단문 → **약한 종합** (개선 권장)
  - 빈 줄 + 50자 이상 일반 문단 → 정상

### #2 정보 진전 정체
- 인접 2~3 문단의 핵심 명사·동사·수치 중복 검토
- 같은 주장이 다른 표현으로 두 번 등장하면 플래그
- **휴리스틱 약함 — Claude 정성 평가 위주**

### #3 표·다이어그램 뒤 공허한 요약
- 종료 직후 첫 문단을 검사할 대상:
  - ` ```mermaid ... ``` ` 코드 블록
  - 마크다운 표 (`|---|` 헤더 구분선)
  - `<svg>` / `<table>` / `<div class="diagram-...">` HTML 블록
- 다음 표지 표현이 등장하면 의심:
  - "이 모든 것이", "위와 같이", "정리하면", "이 N가지가", "전체적으로 보면"
- 그 문단이 (a) 결정적 항목, (b) 빠지면 무너지는 의존성, (c) 흔한 오해 중 어느 것도 짚지 않으면 **안티패턴**

### #4 모호한 지시대명사
- regex 매칭: `(이|그|이러한|이런|그런)\s*(것|상황|경우|케이스|방식|패턴|점|문제|구조)`
- 매칭 위치 위로 1~2 문장 안에 명확한 명사 referent가 있는지 확인
- 없거나 불명확하면 **플래그**, 명사로 재호명 권장

### #5 헤지 vs 1인칭
- 헤지 표현 카운트: "할 수 있다", "할 수도 있다", "인 경향이 있다", "라고 볼 수 있다", "할 가능성이 있다", "일 수 있다"
- 1인칭/경험 표현 카운트: "저는", "제가", "직접", "측정해보니", "운영 중", "본가에서", "후쿠오카에서", "실제로 해보니"
- 판정:
  - 헤지 ≥ 1인칭 × 2 → **전반적 입장 약화**
  - 측정값(숫자+단위)이 같은 문단에 있는데 헤지를 쓰면 → **개별 라인 플래그** (단정형 전환 권장)

### #6 단조로운 리듬
- 5개 이상 항목 묶음에서 각 항목 글자 수 측정
- 변동계수(std / mean) < 0.2 → **단조 플래그**
- 4개 이하 묶음은 무시 (적은 표본은 무의미)

## 출력 형식

다음 마크다운 형식으로 보고한다:

```markdown
# 포스트 점검: {파일명}

대상: `_posts/{path}` ({총 라인 수}줄)

## 🔴 높음 (즉시 수정 권장)

### [패턴 #N] 제목 (M건)
- **L{시작}-{끝}**: 한 줄 요약
  > 인용 (실제 본문에서 1-2줄)
  → 권장: 구체적 수정 방향 (예: "이 13개 항목 중 가장 핵심인 X를 본문 끝에 짚기")

## 🟡 중간 (검토 후 판단)

### [패턴 #N] 제목 (M건)
...

## 🟢 낮음 (취향)
...

## 강점 (유지할 것)

- **L{N}**: 실측값 "X mbps" 잘 사용
- **L{M}**: 1인칭 경험 "후쿠오카에서 측정" 그대로 유지

## 종합 — 상위 3개 우선순위

1. **{패턴 이름}** ({건수}건): 한 문장 요약
2. ...
3. ...

자동 수정은 하지 않았다. 어느 항목을 손볼지 알려주면 그 부분만 수정한다.
```

## 핵심 원칙

- **자동 수정 금지**: 사용자가 결정. 휴리스틱이 잡은 것이 진짜 문제가 아닐 수도 있다
- **강점도 보고**: 무엇을 유지할지 알아야 균형 잡힘. 실측값·1인칭·RFC 인용은 사용자의 강점
- **우선순위 매김**: 모든 발견을 평등하게 나열하지 말고 🔴/🟡/🟢로 구분
- **정성 판단 명시**: 휴리스틱이 약한 #2, #6은 "Claude 정성 평가"라고 표시
- **거짓 양성 인정**: 의도적인 반복(강조용)이나 의도적 단조 리듬(가독성)은 무시할 수 있다고 권장에 명시
